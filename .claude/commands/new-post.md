# 블로그 포스트 작성 모드

주제: $ARGUMENTS

## 실행 순서

### 1단계 — 사전 확인
아래 파일들을 읽는다:
- `.claude/memory/학습진행도.md` — 관련 학습 내용 파악
- `.claude/memory/블로그현황.md` — 기존 포스트 확인 (중복 방지)
- `.claude/memory/글쓰기스타일.md` — 글쓰기 스타일 숙지

---

### 2단계 — 초안 작성
CLAUDE.md의 **글쓰기 원칙** 을 철저히 따른다.

**파일명**: `YYYY-MM-DD-{영어-제목}.md` (오늘 날짜 사용)

**AI 느낌 제거 체크리스트** (작성 전 반드시 확인):
- [ ] 1인칭 구어체 ("했다", "느꼈다", "삽질했다")
- [ ] 실제 경험처럼 구체적 상황 묘사
- [ ] "본 포스트에서는", "알아보겠습니다" 같은 표현 없음
- [ ] 리스트 남발 없이 자연스러운 문장 흐름
- [ ] 짧은 문단 (4~5줄 이하)
- [ ] 제목이 경험 중심

**글 구조 권장**:
1. 도입부 — 이걸 왜 쓰게 됐는지 상황 설명
2. 본문 — 과정 중심 서술 (삽질 포함)
3. 마무리 — 배운 점, 다음 계획 또는 한 줄 소감

---

### 3단계 — 검토 요청
초안 작성 완료 후:

```
📝 초안을 작성했어요. 검토해주세요.

[전체 내용 출력]

---
수정할 부분이 있으면 말씀해주세요.
없으면 "승인" 이라고 입력해주세요.
```

수정 요청이 오면 반영 후 다시 보여준다.

---

### 4단계 — 승인 후 처리
"승인" 입력 시:
1. `_posts/YYYY-MM-DD-제목.md` 로 파일 저장
2. `.claude/memory/블로그현황.md` 업데이트 (초안 → 저장 완료로 변경)
3. 안내 메시지 출력:
   ```
   ✅ 파일이 저장됐어요!
   📁 위치: _posts/YYYY-MM-DD-제목.md
   
   배포하려면 /deploy 를 입력하세요.
   ```
