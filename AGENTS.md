### 테마 (2026-07-17 전면 교체)

Chirpy 테마를 제거하고 자체 테마를 직접 구축했다. 테마 gem이 없으므로
`_layouts`·`_sass`·`_includes`를 직접 수정한다.

- **토큰**: 색은 전부 `_sass/_tokens.scss`의 CSS 변수를 참조한다. **hex 하드코딩 금지.**
  다크모드는 `[data-theme="dark"]`에서 같은 토큰의 값만 재정의한다 — 컴포넌트 SCSS는 건드리지 않는다.
- **import 순서 의존**: `assets/css/main.scss`의 `@import` 순서를 재정렬하면 레이아웃이 조용히 깨진다.
- **구조**: 히어로 + 피처드 3열 + 좌썸네일 리스트 + 태그 사이드바(홈).
  카테고리 전용 페이지는 없다(리스트 라벨로만 노출). 탐색은 태그가 담당한다.
- **기능**: giscus 댓글, GA·GoatCounter(프로덕션 빌드만), 포스트 TOC, 검색(`search.json`), 다크모드.
  PWA는 폐기했다.
- 설계·계획 문서: `docs/superpowers/specs|plans/2026-07-16-blog-redesign*.md`

### 디렉터리 구조
```
_posts/          ← 블로그 포스트 (YYYY-MM-DD-제목.md)
_layouts/        ← default, home, post, tag, page
_includes/       ← header, hero, card-featured, list-item, tag-sidebar, toc, comments, search, footer
_sass/           ← _tokens.scss(토큰) 외 컴포넌트별 파티셜
_tabs/           ← about.md (archives·categories·tags는 테마 교체 때 삭제됨)
assets/img/posts/{글-슬러그}/   ← 포스트 이미지 + thumbnail.webp
.claude/
  commands/      ← 슬래시 커맨드 정의 (deploy, new-post, status, teach)
  memory/        ← 세션 간 진행 상황 기록 (Codex도 이 경로를 쓴다)
  materials/     ← 학습 원본 자료
```

### 포스트 Front Matter 형식
```yaml
---
title: "제목"
date: YYYY-MM-DD HH:MM:SS +0900
categories: [카테고리]
tags: [태그1, 태그2]
image:
  path: /assets/img/posts/{글-슬러그}/thumbnail.webp
  alt: 썸네일이 무엇을 보여주는지 서술
---
```

`image`가 없으면 목록에서 제목만 있는 폴백 카드로 표시된다. 새 글에는 반드시 넣는다.

### 썸네일 규칙

- **3:2 비율, 1200×800**, WebP 저장 (`thumbnail.webp`)
- PNG로 만들었다면 반드시 WebP로 변환한다. PNG 그대로 두면 장당 1~2MB로 비대해진다
  (26장 기준 37MB → 1.8MB로 줄었던 전례가 있다)
- **매체를 섞는다.** 전부 같은 스타일이면 자동 생성 티가 난다.
  노트 볼펜 / 화이트보드 마커 / 연필 스케치 / 인쇄물+형광펜 / 책 페이지 / 터미널·에디터 / 포스트잇
- **그 글에만 있는 실제 숫자·명령어를 넣는다.** 범용 이미지는 성의 없어 보인다
- **화면(터미널·에디터)을 그릴 때는 실제 사양과 대조한다.** 지어낸 로그·버전·에러 코드 금지
- **이미지에 한글을 넣지 않는다.** 생성 시 깨진다. 영문·숫자·기호만

## 🌿 브랜치 전략

```
main       ← 배포 브랜치 (직접 커밋 금지)
  └─ develop  ← 작업 브랜치 (글 작성, 수정, 학습 정리 전부 여기서)
```

**develop → main 병합 = 발행.** 브랜치는 항상 2개만 유지한다. 글마다 브랜치를 만들지 않는다.

배포는 GitHub Actions가 main push 시 자동 수행한다. **`html-proofer` 검사를 통과해야 배포된다** —
프로토콜 상대 URL(`//example.com/x.js`)은 거부되므로 항상 명시적 `https://`를 쓴다.

## ✍️ 글쓰기 원칙 (AI 느낌 제거)

- **1인칭 구어체**: "했다", "느꼈다", "삽질했다", "막혔다"
- **실제 경험처럼** 구체적 상황 묘사 (언제, 어디서, 어떤 오류)
- **완벽한 설명보다** 솔직한 경험 위주
- **짧은 문단**, 적절한 줄바꿈
- 리스트 남발 금지 → 자연스러운 문장 흐름
- "저는", "본 포스트에서는" 같은 딱딱한 표현 금지

### 좋은 예 ✅
> "Redis를 처음 붙여봤는데 Windows에서 설치하다가 막혔다. 포트가 이미 점유돼 있었는데 이걸 찾는 데만 30분 날렸다."

### 나쁜 예 ❌
> "본 포스트에서는 Redis의 설치 방법에 대해 알아보겠습니다."

### 제목 스타일
