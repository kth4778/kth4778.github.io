# 블로그를 시작하게 된 이유

개발 공부를 하면서 항상 느꼈던 건:

"분명 이해했다고 생각했는데 시간이 지나면 다시 까먹는다"

는 점이었다.

특히:

- Spring
- Redis
- 트러블슈팅
- 프로젝트 설계

같은 것들은 직접 기록하지 않으면 금방 흐려졌다.

그래서 단순 암기보다:

"기록하면서 성장하자"

라는 생각으로 기술 블로그를 시작하게 되었다.

---

# 왜 GitHub 블로그를 선택했는가

처음에는 Velog도 고민했다.

하지만:

- Git 기반 관리
- Markdown 작성
- 개발자다운 커스터마이징
- 포트폴리오 느낌

이 마음에 들어 GitHub Pages + Jekyll + Chirpy 조합을 선택했다.

다만 Windows 환경에서 Ruby 버전 문제로 꽤 삽질을 했다.

특히 Ruby 4.x 버전이 Jekyll과 충돌해서 결국 Ruby 3.3.x로 다시 설치해야 했다.

그래도 하나씩 해결하면서 개발 블로그 구조를 이해할 수 있었다.

---

# 앞으로 기록하고 싶은 것들

앞으로는 아래 내용들을 꾸준히 기록하려고 한다.

- SW마에스트로 활동
- 팀 프로젝트 경험
- Spring / Backend 기술 정리
- 트러블슈팅 기록
- 코딩테스트 풀이
- CS 공부 내용
- 회고 및 성장 기록

특히 단순 개념 정리보다:

"왜 이런 선택을 했는가"

를 많이 남기고 싶다.

---

# 블로그 방향

이 블로그는:

"백엔드 개발자로 성장하는 과정을 기록하는 공간"

이 되었으면 좋겠다.

완벽한 글보다:

- 직접 겪은 경험
- 삽질
- 고민 과정
- 해결 과정

들을 솔직하게 남겨보려고 한다.

"이때는 이런 고민을 했었구나"
