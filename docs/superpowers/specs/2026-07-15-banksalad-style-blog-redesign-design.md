# 블로그 디자인 개편 — 뱅크샐러드 기술블로그 스타일 (설계안)

작성일: 2026-07-15
작성자: Claude (brainstorming 세션), 승인자: 김태현

## 배경 및 목표

현재 블로그는 Jekyll + Chirpy 테마(gem 의존, `jekyll-theme-chirpy ~> 7.5`)로 운영 중이다. 기존 Chirpy 디자인을 사이트 전체에서 걷어내고, 뱅크샐러드 기술 블로그(`blog.banksalad.com`)의 디자인 언어를 참고해 새로 만든다. 1차로 뱅크샐러드 스타일을 최대한 가깝게 구현하고, 완성 후 사용자가 원하는 개인화 디자인 요소를 추가로 얹는 2단계 진행을 전제로 한다.

## 레퍼런스 조사 결과 (blog.banksalad.com)

- 폰트: Pretendard (한국어 산세리프)
- 타이틀: 48px, font-weight 500 (미디엄, 볼드 아님)
- 색상: 절제된 흑백/그레이 톤 — 활성 탭 `#292929`, 비활성 탭 `#757575`, 포인트 컬러 거의 없음
- 상단 내비게이션: 얕은 탭 구조 (Tech / Culture / Banksalad 3개)
- 포스트 카드: 날짜 배지("16 JAN" 형태, 일/월 2줄) + 해시태그형 태그 + 제목 + 한 줄 설명 + "Read More" 링크로 구성된 리스트형 레이아웃
- 페이지네이션: "1/4" 형태의 심플한 표기
- 전체적으로 다크모드 토글, 댓글 위젯, PWA 설치 배너 등은 노출되지 않음 (심플한 콘텐츠 중심 레이아웃)

- 홈 (포스트 리스트)
- 포스트 상세
- Categories

- Giscus 댓글
- PWA (오프라인 설치)

## 아키텍처

Chirpy는 Gemfile에 gem으로 설치되어 있어 레이아웃/인클루드/스타일 대부분이 gem 내부에 있다. "기존 디자인 제거"는 실질적으로 아래 두 가지를 의미한다.

1. Gemfile에서 `jekyll-theme-chirpy` 의존성을 제거하고, 순수 Jekyll(테마 없는 최소 구성)로 전환
2. 리포 안에 `_layouts/`, `_includes/`, `_sass/` 또는 `assets/css/`를 전부 새로 작성해 모든 페이지를 커스텀 테마로 렌더링

Chirpy가 내부적으로 의존하던 부가 gem(jekyll-paginate, jekyll-archives, jekyll-seo-tag 등 페이지네이션/아카이브/SEO에 필요한 것들)은 유지가 필요한 것만 Gemfile에 직접 추가한다.

**디자인 생성 도구:** UI/UX Pro Max 스킬(설치 완료, v2.11.0)을 구현 단계에서 호출해 뱅크샐러드 레퍼런스 기반 컬러 팔레트·컴포넌트 스타일을 생성한다. 색상·간격 값은 CSS 커스텀 프로퍼티(변수)로 구조화해서 추후 사용자가 포인트 컬러 등 개인화 요소를 얹기 쉽게 한다.

## 페이지 인벤토리

| 페이지 | 내용 |
|---|---|
| 홈 | 날짜 배지 + 태그 + 제목 + 한줄 설명 + Read More 카드형 리스트, 페이지네이션 |
| 포스트 상세 | 제목/날짜/태그 + 본문 (댓글·다크모드 없음) |
| Categories | 카테고리별 리스트 (뱅크샐러드 Tech/Culture 탭 컨셉 확장 — 백엔드/AI/책 Log 등 기존 카테고리 구조 유지) |
| Tags | 태그별 리스트 |
| Archives | 연도·월별 아카이브 |
| About | 미니멀 텍스트 소개 페이지 |

## 보존 vs 제거 파일

| 구분 | 대상 |
|---|---|
| **보존** | `_posts/**`, `assets/img/**`, `.env`, `_data/**`, `.claude/**`(커맨드·메모리 전부), `CLAUDE.md` |
| **제거 후 재작성** | Chirpy 관련 레이아웃/인클루드/스타일 오버라이드, `assets/css`, Chirpy 테마 전용 JS, Gemfile의 chirpy 의존성 |

기존 포스트 마크다운의 프론트매터(title/date/categories/tags)는 그대로 재사용되므로 글 마이그레이션은 불필요하다.

플러그인(UI/UX Pro Max, Superpowers 등)은 리포가 아닌 사용자 홈 디렉터리(`~/.claude/plugins/`)에 설치되어 있어 이번 작업과 무관하다.

## 전환 순서 (리스크 관리)

Jekyll은 레이아웃 파일이 없으면 빌드가 깨진다. 따라서 구현은 다음 순서를 따른다.

1. 새 레이아웃/인클루드/스타일을 리포에 전부 작성
2. 로컬 Jekyll 서버로 전체 페이지 유형 렌더링 검증
3. 문제없음을 확인한 뒤에만 Chirpy gem 의존성을 Gemfile에서 제거

먼저 지우고 나중에 새로 만드는 순서로 가면 그 사이 사이트 전체가 깨진 상태가 되므로 반드시 이 순서를 지킨다. 세부 작업 단계는 writing-plans 단계에서 쪼갠다.

## 검증 계획

- 로컬 Jekyll 서버(`bundle exec jekyll serve`)로 6개 페이지 유형 렌더링 확인
- html-proofer로 깨진 링크/이미지 검사
- 모바일 반응형 확인
- 기존 포스트 23편 + 초안 1편이 모두 정상 렌더링되는지 확인
